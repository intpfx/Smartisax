## Build Notes For V0.5

Prefer a new `system-otatrust-v0.5-debloat.img` based on the current successful
v0.4 image, not the older v0.2 baseline.

For `/product/overlay` removals, either:

```text
1. Patch product_b as well as system_b in super, or
2. Defer overlay removals until the build script supports exact-current
   product_b replacement.
```

For updated-system packages currently under `/data/app`, do not rely on ROM
removal alone. Plan a post-boot data cleanup/verification step similar to the
v0.4 launcher shortcut dry-run workflow.
